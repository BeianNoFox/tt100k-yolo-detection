# 基于YOLOv8的交通标志检测研究

## 一、论文（设计）选题的目的和意义

交通标志检测是计算机视觉中的一项基础任务，主要应用于自动驾驶、辅助驾驶等场景。中国道路交通标志种类较多，检测场景复杂，对检测算法的精度有一定要求。

本课题选用TT100K数据集作为实验基础。该数据集包含约10万张中国城市街景图片与221类交通标志标注，数据来源于腾讯街景采集。对数据集的初步统计表明：其一，约42%的标志目标在原始图像中尺寸小于32×32像素，属于小目标范畴，若将高分辨率原图直接缩放至标准输入，小标志的细节信息将丢失；其二，类别分布不均匀，超过80%的类别实例数低于100，部分类别仅有个位数样本。

针对上述情况，本课题使用YOLOv8作为基础检测模型，通过提高输入分辨率、对低频类别进行重复采样、加强数据增强等方式进行改进，并通过消融实验验证各措施的效果。同时，课题将数据预处理、模型训练、评估分析和推理服务整合为一个完整的检测系统。

## 二、国内外关于该论题的研究现状和发展趋势

研究现状：

目标检测方法在深度学习的推动下经历了较快发展。R-CNN系列的提出为目标检测奠定了基础，此后Faster R-CNN、SSD等方法的出现进一步完善了检测流程。YOLO系列作为单阶段检测方法，以端到端的检测方式在工程实践中得到较多应用，迭代至今已有多个版本。YOLOv8作为较新的版本，集成了无锚框检测头和多种数据增强策略，提供了完整的训练与部署工具链。

针对小目标检测问题，现有研究主要从三个方向入手：一是提高模型输入分辨率，使模型直接接收更大尺寸的输入图像；二是采用切片策略，将大图切分为若干子图分别检测后进行结果合并；三是从模型结构角度出发，引入更高分辨率的浅层特征图以增强小目标表征能力。

针对类别不平衡问题，常用方法包括Focal Loss、类别加权损失和重复采样等。Focal Loss通过降低易分类样本的损失权重，使模型更多地关注难分样本；类别加权损失为不同类别设置不同的损失权重；重复采样直接对样本量不足的类别进行复制扩充。Mosaic和MixUp等数据增强方法除增加样本多样性外，也能在一定程度上缓解类别不平衡。

TT100K数据集由清华大学和腾讯于2016年联合发布，图像来源为腾讯街景，标注涵盖221类中国交通标志，规模较大且场景多样。由于类别间样本量差异较大，现有研究在实际训练中通常筛选实例数不少于100的类别（约45类）作为子集进行训练。

发展趋势：

在模型轻量化方面，相关研究致力于在保持检测精度的同时降低模型体积，以适应边缘设备的部署需求；在场景泛化方面，部分工作开始关注恶劣天气、低照度等复杂条件下的检测鲁棒性；在数据高效利用方面，半监督学习和少样本学习等方法有助于减轻对大规模标注数据的依赖。

## 三、论文（设计）的主攻方向、主要内容、研究方法及技术路线

### 1. 主攻方向

（1）TT100K数据集的预处理
实现TT100K数据集标注格式的解析与转换，完成类别筛选与分层数据集划分。

（2）面向小目标与类别不均衡的训练策略
针对小目标问题，研究输入分辨率提升对检测精度的影响；针对类别不均衡，研究加倍采样与数据增强策略对中低频类别检测效果的提升。

（3）检测系统的构建
将数据预处理、模型训练、评估分析及推理服务整合为可运行系统。

### 2. 主要内容

（1）数据预处理模块
实现对TT100K数据集标注文件的解析，统计类别实例数与目标尺寸分布。基于实例数阈值（≥100）筛选得到约45类的训练子集，完成JSON到YOLO格式的标注转换，采用分层采样按7:2:1划分为训练集、验证集和测试集。针对长尾问题，实现对低频类别的图片加倍采样。

（2）消融实验设计
设置四组递进实验：
- 实验1（基线）：YOLOv8s，1024×1024输入分辨率，不加额外处理
- 实验2（切片检测）：对原图进行640×640切片，对比0%、20%、30%三组重叠率
- 实验3（长尾处理）：在实验1基础上引入加倍采样（低频类按样本数据分档复制3-10倍），加强Mosaic和MixUp增强
- 实验4（综合优化）：整合前序实验中的最优策略并调参

（3）模型训练与评估
使用YOLOv8s模型，以COCO预训练权重初始化。评估指标包括mAP@0.5、mAP@0.5:0.95及各分类AP值，按类别频次和小目标尺寸分桶进行分析。

（4）推理服务
基于Flask构建REST API，支持图片上传检测，返回目标类别、置信度与边界框坐标。

### 3. 研究方法

（1）实验对比
采用消融实验设计，逐组验证不同改进措施的效果，以mAP为主要评估指标。

（2）数据处理
对低频类别按样本数分档进行重复采样（＜50实例→10倍，50-100→5倍，100-200→3倍），结合Mosaic和MixUp增强。

（3）结果分析
从整体指标、类别分层、目标尺寸分桶三个维度分析实验结果，使用matplotlib生成对比图表。

### 4. 技术路线

系统使用Python开发，主要依赖PyTorch和Ultralytics框架。整体分为三层：

- 数据与训练层：TT100K数据预处理（统计、筛选、格式转换、数据集划分、加倍采样），YOLOv8s模型训练与评估。
- 推理服务层：Flask框架提供API接口，加载训练模型进行推理。
- 数据处理流程：原始JSON标注→类别筛选→YOLO格式转换→数据集划分→加倍采样（实验3/4）→YOLOv8s训练→模型评估→推理API。
